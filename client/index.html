Status Guidelines

  • 200 - OK: The request was successful. The server has fulfilled the request and is returning the content.
  • 301 - Moved Permanently: The requested resource has been permanently moved to a new location. The response should contain a "Location" header with the new URL. Search engines will update their links to the new location.
  • 302 - Found: The requested resource resides temporarily under a different URI. The response should contain a "Location" header with the new URL. Search engines might not update their links to the new location (important for SEO).
  • 307 - Temporary Redirect: The requested resource resides temporarily under a different URI. Unlike 302, this code specifies that the original request method (GET, POST, etc.) should be preserved. Search engines might not update their links to the new location.
  • 403 - Forbidden: The server understood the request but is refusing to fulfill it. Authorization is required, and the client did not provide the proper credentials.
  • 404 - Not Found: The requested resource could not be found but may be available again in the future
  • 500 - Internal Server Error: The server encountered an unexpected condition that prevented it from fulfilling the request.
  • 999 - Unknown Code: This code is not an official HTTP status code. Its meaning depends on the specific server implementation. It's generally recommended to use established codes for better compatibility,
